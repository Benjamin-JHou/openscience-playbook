# Contributing to the AI-Scientist Playbook

Thank you for your interest in contributing to the **AI-Scientist Playbook**! We welcome contributions from researchers, software developers, AI practitioners, and students. By contributing, you help make AI-driven scientific discovery more accessible and robust for everyone.

---

## How You Can Contribute

### 1. Adding or Updating AI Scientist Agents
If you discover a new AI agent, workspace, or framework for scientific research that is not listed in our matrices:
- Update the **AI-Scientist Resource Matrix** in all language READMEs with the project name, developer, repository/website links, tech stack, and target domain.
- Add a detailed profile under **Core Project Profiles** if the project is open-source and ready for local deployment.
- Provide its details in the **Evo-Map / Ecosystem Landscape** table.

### 2. Sharing Custom Skills & MCP Connectors
We want to expand our list of vertical domain skills and MCP servers:
- Share your custom Model Context Protocol (MCP) servers or open-agent-skills packages (e.g. bioinformatics pipelines, chemical informatics datasets, geospatial tools).
- Document them under the **Extended Skills Library & MCP Extensions** section.

### 3. Adding Advanced Tips & Case Studies
- Have you optimized an AI agent workflow for a specific journal's submission?
- Did you find a clever way to reduce API call costs using specific model routing strategies?
- Share your experience and add them to the **Advanced Tips & Tricks** section.

### 4. Improving Translations
This playbook is available in English, Chinese, French, Japanese, Korean, and Spanish. If you notice any awkward translations, grammatical errors, or missing technical terms, please submit a pull request targeting the specific language README (e.g., `README_zh.md`, `README_fr.md`, etc.).

---

## Submission Guidelines

1. **Fork the Repository**: Create a personal fork of the repository on GitHub.
2. **Create a Branch**: Use a descriptive branch name (e.g., `feature/add-new-mcp-connector` or `fix/french-translation`).
3. **Commit Changes**: Make clear, atomic commits with informative messages.
4. **Link Integrity**: Ensure all local file links (using relative paths) and external URLs resolve correctly. Do not use absolute paths.
5. **Open a Pull Request**: Submit your pull request to the `main` branch. Provide a clear summary of what you added or modified.

---

## Code of Conduct & Standards

- **Evidence-Based**: All listed resources and tips should be tested and verified to ensure they resolve to real projects, active repositories, and working databases.
- **Local-First & Open**: We prioritize local-first, model-agnostic, open-source projects that protect researchers' data privacy and foster open science.
- **Respectful Collaboration**: Be respectful and supportive of other contributors and maintainers.

We appreciate your time and expertise in enriching this playbook!
